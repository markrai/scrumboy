# Audit Trail

## Overview

The audit trail provides a durable, append-only log of significant actions within a project. Events are stored in the `audit_events` table and instrumented at the store layer. Assignee history remains in `todo_assignee_events` (separate, specialized).

---

## Action Vocabulary

Canonical past-tense event names:

| Action                               | Target Type | Description                                                                 |
|--------------------------------------|-------------|-----------------------------------------------------------------------------|
| todo_created                         | todo        | Todo created                                                                |
| todo_updated                         | todo        | Title, body, sprint, estimation, or tags changed                             |
| todo_deleted                         | todo        | Todo deleted                                                                |
| todo_moved                           | todo        | Todo moved between columns                                                  |
| member_added                         | member      | Member added to project                                                     |
| member_removed                       | member      | Member removed from project                                                 |
| member_role_changed                  | member      | Member role changed                                                         |
| project_created                      | project     | Project created (durable or temp)                                           |
| project_renamed                      | project     | Project name changed                                                        |
| project_image_updated                | project     | Project image or dominant color changed                                     |
| project_default_sprint_weeks_updated | project     | Default sprint weeks (1 or 2) changed                                       |
| project_deleted                      | project     | Project deleted                                                             |
| link_added                           | todo_link   | Link between todos created                                                  |
| link_removed                         | todo_link   | Link between todos removed                                                  |

---

## Event Model

| Field         | Purpose                                                                 |
|---------------|-------------------------------------------------------------------------|
| actor_user_id | Who performed the action; NULL for anonymous                            |
| action        | Verb from canonical list above                                          |
| target_type   | Entity: todo, project, member, todo_link                               |
| target_id     | Entity ID. For project: project_id. For todo: todo.id. For member: user_id. For todo_link: NULL |
| project_id    | Project scope                                                           |
| metadata      | JSON with action-specific data (marshaled from Go)                      |
| created_at    | Timestamp (Unix ms). Generated by Go `nowUnixMs()`; not SQLite          |

---

## target_id Rules

- **Project events:** `target_type='project'` and `target_id=project_id`
- **Todo, member:** Entity ID
- **todo_link:** `target_id=NULL` (no stable row ID); link details in metadata

---

## Metadata

JSON is generated from Go structs; no hand-rolled strings. Examples:

- **todo_created:** `{"local_id": 1, "column_key": "backlog", "title": "..."}` — full title stored
- **todo_updated:** `{"changed_fields": ["body"], "before_len": 120, "after_len": 240}` or `{"tags_added": ["bug"], "tags_removed": ["enhancement"]}` — no full body/title content
- **todo_moved:** `{"from_column": "backlog", "to_column": "doing", "local_id": 1}`
- **member_added:** `{"user_id": 5, "role": "contributor"}`
- **member_role_changed:** `{"user_id": 5, "from_role": "contributor", "to_role": "maintainer"}`
- **project_renamed:** `{"from_name": "Old", "to_name": "New"}`
- **link_added / link_removed:** `{"from_todo_id": 1, "to_todo_id": 2, "from_local_id": 1, "to_local_id": 2, "link_type": "blocks"}`

---

## Storage

- **Table:** `audit_events`
- **Append-only:** Triggers prevent UPDATE and DELETE
- **Indexes:** `project_id`, `action`, `actor_user_id` (with `created_at` for efficient queries)

---

## Assignee History

Assignee changes are tracked in `todo_assignee_events` only. They are not duplicated in `audit_events`.

---

## Integration Points

Store methods that write audit events:

| Store Method                              | Action                               |
|-------------------------------------------|--------------------------------------|
| CreateTodo                                | todo_created                        |
| UpdateTodo                                | todo_updated (when fields changed)  |
| DeleteTodo                                | todo_deleted                         |
| MoveTodo                                  | todo_moved                           |
| AddProjectMember                          | member_added                         |
| RemoveProjectMember                       | member_removed                       |
| UpdateProjectMemberRole                   | member_role_changed                  |
| CreateProject / CreateProjectWithWorkflow | project_created                      |
| CreateAnonymousBoard                      | project_created                      |
| UpdateProjectName                         | project_renamed                      |
| UpdateProjectImage                        | project_image_updated                |
| UpdateProjectDefaultSprintWeeks           | project_default_sprint_weeks_updated |
| DeleteProject                             | project_deleted                      |
| AddLink                                   | link_added                           |
| RemoveLink                                | link_removed                         |

---

## Security

- **Immutability:** Append-only triggers prevent tampering
- **No PII in metadata:** Default metadata avoids full content; body/title use lengths only
- **Actor identification:** `actor_user_id` is NULL for anonymous boards
